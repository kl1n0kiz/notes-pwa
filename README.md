Notes PWA

Прогрессивное веб-приложение (PWA) для создания, редактирования и управления заметками.  
Разработано в рамках практических занятий 13–17 по дисциплине **«Фронтенд и бэкенд разработка»**.


Содержание практических занятий

Практика 13 — Основы PWA. Web App Manifest

- Создана базовая структура приложения для заметок
- Добавлен файл `manifest.json` с описанием приложения:
  - `name`, `short_name`, `description`
  - иконки нескольких размеров (192×192, 512×512)
  - `start_url`, `display: standalone`, `theme_color`, `background_color`
- Приложение доступно для установки на главный экран устройства

Практика 14 — Service Worker. Кэширование ресурсов

- Зарегистрирован Service Worker (`service-worker.js`)
- Реализовано кэширование статических ресурсов (HTML, CSS, JS, иконки) при событии `install`
- Настроена стратегия **Cache First** — приложение работает офлайн
- Обработка события `activate` с очисткой устаревшего кэша

Практика 15 — HTTPS и деплой приложения

- Приложение развёрнуто по протоколу **HTTPS** (обязательное требование для PWA)
- Продемонстрирована работа сайта через защищённое соединение
- Проверена корректность регистрации Service Worker в DevTools (вкладка Application)

Практика 16 — Push-уведомления

- Реализована подписка пользователя на push-уведомления через **Push API**
- Использован `PushManager.subscribe()` с VAPID-ключами
- На стороне Service Worker обработано событие `push` — при получении уведомления отображается `showNotification()`
- Реализована отправка тестового push-уведомления

Практика 17 — UI уведомлений и доработка интерфейса

- Настроен внешний вид уведомлений: заголовок, тело сообщения, иконка
- Добавлена обработка клика по уведомлению (`notificationclick`) — открытие/фокус вкладки приложения
- Улучшен пользовательский интерфейс:
  - Кнопка подписки/отписки от уведомлений с визуальной индикацией статуса
  - Отображение состояния разрешения (`granted` / `denied` / `default`)
- Проверена видимость уведомлений в различных браузерах

---

Технологии

| Технология | Назначение |
|---|---|
| HTML5 / CSS3 / JavaScript | Основа приложения |
| Web App Manifest | Установка PWA на устройство |
| Service Worker | Офлайн-режим, кэширование |
| Cache API | Хранение статических ресурсов |
| Push API / Notifications API | Push-уведомления |
| HTTPS | Безопасное соединение (требование PWA) |
| VAPID | Аутентификация push-сервера |

---

Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/kl1n0kiz/notes-pwa.git
cd notes-pwa

# Открыть index.html через локальный HTTPS-сервер
# Например, с помощью live-server или VS Code Live Server
```

> ⚠️ Service Worker и Push API работают **только по HTTPS** (или на `localhost`).

Проверка PWA в браузере

1. Открыть DevTools → вкладка **Application**
2. **Manifest** — проверить корректность манифеста и иконок
3. **Service Workers** — убедиться, что воркер зарегистрирован и активен
4. **Cache Storage** — просмотреть закэшированные ресурсы
5. **Push Messaging** — протестировать отправку уведомлений
